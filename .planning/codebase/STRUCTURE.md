## Naming Conventions

**Files:**
- Route handlers: lowercase, single word or dash-separated. Examples: `calendar.js`, `payment-info.js`, `quote.js`
- Utility modules: lowercase, descriptive name. Examples: `guesty.js`, `upsells-config.js`, `notify.js`
- Single responsibility — one route per file

**Directories:**
- Top-level: lowercase plural or singular. Examples: `api/`, `lib/`
- Vercel convention: `api/` directory automatically maps to `/api/` routes
- No nested subdirectories in `api/` (Vercel limitation — all routes must be flat)

**Function Names:**
- camelCase. Examples: `getToken()`, `fetchAvailability()`, `createQuote()`
- Async functions: prefix with `async` keyword (example: `async function handler(req, res)`)
- Private helpers: prefix with `_` if needed (example: `_formatDate()`)

**Variables:**
- camelCase for all variables and constants. Examples: `cachedToken`, `tokenExpiry`, `listingId`, `startDate`
- Environment variables: UPPER_SNAKE_CASE. Examples: `GUESTY_CLIENT_ID`, `GUESTY_LISTING_ID`

**Response Fields:**
- camelCase in JSON. Examples: `{ checkIn, checkOut, quoteId, priceRange, avgPrice }`
- Match Guesty BEAPI field names where applicable (document any transformations)

## Where to Add New Code

**New API Endpoint:**
1. Create file in `api/` with name matching route path. Example: `api/new-feature.js`
2. Export default async handler: `export default async function handler(req, res) { ... }`
3. Add CORS headers and OPTIONS handling (copy from `api/calendar.js` lines 23-26)
4. Get token via `getToken()` call (will move to `lib/guesty.js`)
5. Make Guesty API call
6. Transform response
7. Return `res.status(200).json({ ... })`
8. Wrap in try-catch for error handling (example: `api/calendar.js` lines 58-61)

**New Utility/Helper:**
1. Create file in `lib/` with descriptive name. Example: `lib/email-formatter.js`
2. Export functions as named exports: `export async function sendEmail(to, subject, body) { ... }`
3. Import in route handlers: `import { sendEmail } from '../lib/email-formatter.js'`
4. Keep utilities focused and testable

**New Configuration:**
1. If Upsell-related: add to `lib/upsells-config.js`
2. If Guesty-related: add to environment variables in Vercel dashboard (see vercel.json section)
3. If CORS/Headers related: add to `vercel.json` headers array (lines 1-3)

## Vercel Deployment

**vercel.json (lines 1-3):**
- `headers` array: Applies headers to routes matching `source` regex
- `"/api/(.*)"` regex matches all routes under `/api/`
- Applied headers: CORS allow-origin, allow-methods, cache control
- Cache rule: `s-maxage=3600` (1 hour server cache), `stale-while-revalidate=7200` (2 hour stale cache)

**Environment Variables (Vercel dashboard):**
- `GUESTY_CLIENT_ID`: OAuth client ID (required)
- `GUESTY_CLIENT_SECRET`: OAuth client secret (required, never log)
- `GUESTY_LISTING_ID`: Hardcoded listing ID after discovery (required)
- Future: `STRIPE_PK` (frontend-safe, can be committed)
- Future: `SEBASTIAN_EMAIL` if email notifications added
